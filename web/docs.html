系统概述

BVC（Binary Vulnerability Detection）研究平台是一个专门用于二进制代码漏洞检测的学术研究工具。 本平台集成了多种先进的深度学习模型，包括BERT、VulSeeker、SAFE和SENSE， 用于分析和比较不同模型在各种二进制程序上的漏洞检测性能。

主要功能

  • 实验结果可视化：查看各模型的性能对比图表和ROC曲线
  • 交互式数据分析：筛选、排序和统计分析实验数据
  • 性能指标对比：AUC、精确率、召回率、F1-Score等多维度比较
  • 数据导出功能：支持CSV格式的数据导出

使用流程

  1. 项目概述：了解研究背景、目标和支持的模型
  2. 实验结果：查看性能对比图表和详细的实验数据
  3. 数据分析：使用筛选工具进行深入的数据分析
  4. 使用说明：查看技术文档和参考资料

模型介绍

BERT (Bidirectional Encoder Representations from Transformers)

基于BERT预训练模型的二进制代码表示学习方法。通过对二进制代码进行预处理和特征提取， 利用BERT的双向编码能力学习代码的语义表示，从而实现高精度的漏洞检测。

技术特点：

  • 双向上下文理解
  • 预训练+微调策略
  • 强大的语义表示能力
  • 在多种程序上表现出色

性能表现：

  • 平均AUC: 0.976
  • 最佳配置：Coreutils ARM O1 (AUC: 0.982)
  • 适用场景：所有类型的二进制程序

VulSeeker

专门针对漏洞检测设计的图神经网络模型。通过构建程序的控制流图和数据流图， 使用图卷积网络学习程序的结构特征，实现精确的漏洞定位和检测。

技术特点：

  • 图神经网络架构
  • 控制流和数据流综合分析
  • 结构感知的特征学习
  • 针对漏洞检测优化

性能表现：

  • 平均AUC: 0.959
  • 最佳配置：Busybox ARM O1 (AUC: 0.969)
  • 适用场景：结构复杂的大型程序

SAFE (Static Analysis for Feature Extraction)

基于自注意力机制的二进制代码分析模型。通过静态分析提取程序特征， 使用注意力机制关注关键的代码片段，实现高效的漏洞检测。

技术特点：

  • 自注意力机制
  • 静态特征提取
  • 关键代码片段识别
  • 计算效率高

性能表现：

  • 平均AUC: 0.942
  • 最佳配置：Busybox ARM O1 (AUC: 0.957)
  • 适用场景：快速批量检测

SENSE (Structural Embedding for Neural Software Engineering)

结构感知的二进制代码相似性检测模型。通过学习程序的结构嵌入表示， 实现高效的代码相似性比较和漏洞模式匹配。

技术特点：

  • 结构嵌入学习
  • 相似性比较机制
  • 模式匹配算法
  • 跨架构适应性

性能表现：

  • 平均AUC: 0.925
  • 最佳配置：Busybox ARM O1 (AUC: 0.945)
  • 适用场景：跨平台漏洞检测

使用指南

实验结果页面

实验结果页面提供了各模型性能的可视化展示和详细对比分析。

主要功能：

  • 模型性能总览：显示平均AUC、精确率、召回率和F1-Score
  • 筛选控制：按程序、架构和优化级别筛选数据
  • 性能对比图表：柱状图显示各模型的AUC值
  • ROC曲线对比：直观比较模型的分类性能
  • 详细性能分析：按程序和架构分类的性能分析
  • 训练收敛分析：显示模型训练过程

操作说明：

  1. 使用顶部的下拉菜单选择要分析的程序、架构或优化级别
  2. 图表会自动更新以反映筛选结果
  3. 悬停在图表上查看详细数值
  4. 使用导出按钮下载数据

数据分析页面

数据分析页面提供了更深入的交互式分析工具。

主要功能：

  • 模型筛选：选择要分析的特定模型
  • 指标选择：选择AUC、精确率、召回率等不同指标
  • 统计概览：显示数据点总数、最佳性能等统计信息
  • 分布分析：性能分布直方图和箱线图
  • 相关性分析：不同指标间的相关性热力图
  • 详细数据表格：可搜索、排序的完整数据表
  • 模型对比工具：选择两个模型进行详细对比

操作说明：

  1. 使用顶部的按钮选择要分析的模型和指标
  2. 查看统计概览了解数据的基本情况
  3. 使用搜索框在数据表中快速查找特定配置
  4. 使用排序功能按不同指标排列数据
  5. 使用模型对比工具进行深入的性能比较

数据格式

系统使用的实验数据来源于CSV文件，包含以下指标：

性能指标：

  • AUC：Area Under Curve，ROC曲线下面积
  • 精确率 (Precision)：正确预测的正样本数 / 预测为正的样本数
  • 召回率 (Recall)：正确预测的正样本数 / 实际正样本数
  • F1-Score：精确率和召回率的调和平均数
  • 准确率 (Accuracy)：正确预测的样本数 / 总样本数

实验配置：

  • 程序：coreutils, busybox, openssl
  • 架构：arm, mips, x86
  • 优化级别：O0, O1, O2, O3

性能优化

为了确保良好的用户体验，系统采用了多种性能优化策略：

  • 数据预处理：在客户端预处理和缓存数据
  • 图表复用：避免重复创建图表实例
  • 响应式设计：适配不同屏幕尺寸
  • 异步加载：延迟加载非关键资源
  • 事件防抖：优化频繁触发的事件
